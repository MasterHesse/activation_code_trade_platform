### 1.1 模块说明

#### `actrade-api`
后端 API 工程，基于 Spring Boot 3 + Java 21 开发，当前已完成用户、商家、商品、商品分类、商品详情等基础模块。

#### `actrade-web`
前端工程目录，当前尚未补充具体实现。

#### `documents`
项目文档目录，存放项目计划文档、项目结构文档等说明性文件。

#### `infra`
基础设施目录，当前包含 Docker Compose 配置，用于本地数据库等依赖服务启动。

---

## 2. 后端分层结构说明

后端代码采用按业务模块组织的结构，并在模块内进一步分层：

- `api`：控制器层，对外暴露 REST API
- `api/request`：请求 DTO
- `api/response`：响应 DTO
- `application`：应用服务层，承接业务流程编排
- `domain`：领域模型层，包含实体、枚举等核心业务对象
- `persistence`：持久化层，包含 Repository 与查询规格
- `common`：公共响应模型、公共异常
- `config`：系统配置，如安全配置
- `web`：Web 层统一异常处理等

当前主要业务模块包括：

- `app_users`：用户模块
- `merchant`：商家模块
- `product`：商品模块

## 4. 数据模型设计（Data Model）

当前系统核心数据模型包括：

- 用户 `app_users`
- 商家 `merchant`
- 商品分类 `product_category`
- 商品 `product`
- 商品详情 `product_detail`

这些表共同构成当前交易平台的基础商品管理能力。

---

## 5. 数据库表设计

## 5.1 用户表 `app_users`

| 字段名 | 类型 | 说明 |
|---|---|---|
| user_id | UUID | 主键 |
| username | varchar(64) | 登录账号，唯一 |
| password_hash | varchar(255) | 密码哈希 |
| nickname | varchar(64) | 昵称 |
| email | varchar(128) | 邮箱 |
| phone | varchar(32) | 手机号 |
| role | varchar(32) | 角色 |
| status | varchar(32) | 用户状态 |
| last_login_at | timestamp | 最后登录时间 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

### 索引建议
- `uk_app_users_username`
- `uk_app_users_email`
- `uk_app_users_phone`

---

## 5.2 商家表 `merchant`

| 字段名 | 类型 | 说明 |
|---|---|---|
| merchant_id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| merchant_name | varchar(128) | 商家名称 |
| merchant_type | varchar(32) | 个人/团队/企业 |
| contact_name | varchar(64) | 联系人 |
| contact_email | varchar(128) | 联系邮箱 |
| contact_phone | varchar(32) | 联系电话 |
| license_info | text | 资质/授权说明 |
| audit_status | varchar(32) | 审核状态 |
| audit_remark | text | 审核备注 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

### 说明
- `user_id` 用于关联平台用户
- `license_info` 与 `audit_remark` 使用 `text` 类型存储长文本信息
- 商家审核状态由 `audit_status` 表示，例如：`PENDING`、`APPROVED`、`REJECTED`

---

## 5.3 商品分类表 `product_category`

| 字段名 | 类型 | 说明 |
|---|---|---|
| category_id | UUID | 主键 |
| parent_id | UUID | 父分类 |
| name | varchar(64) | 分类名称 |
| sort_no | int | 排序 |
| status | varchar(32) | 状态 |
| created_at | timestamp | 创建时间 |

### 说明
- 支持通过 `parent_id` 构建分类树
- `sort_no` 用于后台管理排序展示
- `status` 表示分类是否启用

---

## 5.4 商品表 `product`

| 字段名 | 类型 | 说明 |
|---|---|---|
| product_id | UUID | 主键 |
| merchant_id | UUID | 商家 ID |
| category_id | UUID | 分类 ID |
| name | varchar(128) | 商品名称 |
| subtitle | varchar(255) | 副标题 |
| description | text | 简介 |
| cover_image | varchar(255) | 封面图 |
| delivery_mode | varchar(32) | `CODE_STOCK` / `TOOL_EXECUTION` |
| price | numeric(12,2) | 售价 |
| original_price | numeric(12,2) | 原价 |
| status | varchar(32) | 草稿/待审核/上架/下架 |
| stock_count | int | 可用库存快照 |
| sales_count | int | 销量 |
| version_no | int | 乐观锁版本号 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

### 说明
- `description` 使用 `text` 存储商品简介
- `delivery_mode` 区分发货模式：
  - `CODE_STOCK`：卡密库存模式
  - `TOOL_EXECUTION`：工具执行模式
- `stock_count` 对库存模式有效
- 工具模式下 `stock_count` 可以仅作为展示字段
- `version_no` 用于乐观锁控制并发更新

---

## 5.5 商品详情表 `product_detail`

| 字段名 | 类型 | 说明 |
|---|---|---|
| product_id | UUID | 主键 |
| detail_markdown | text | 商品详情 Markdown |
| usage_guide | text | 使用说明 |
| activation_notice | text | 激活注意事项 |
| refund_policy | text | 售后说明 |
| faq_content | text | FAQ |
| updated_at | timestamp | 更新时间 |

### 说明
- 与 `product` 一对一关联
- 各字段均为长文本说明内容，统一采用 `text` 类型
- 适合在后台进行富文本/Markdown 内容维护

## 7. 本地运行说明

### 7.1 启动基础设施

进入 `infra` 目录后，使用 Docker Compose 启动 PostgreSQL 等依赖服务：

```bash
docker compose up -d
```

### 7.2 启动后端服务

进入后端目录：

```bash
cd actrade-api
./mvnw spring-boot:run
```

Windows 下可使用：

```bash
mvnw.cmd spring-boot:run
```

### 7.3 测试建议

- 优先确认数据库连接与表结构初始化成功
- 确认 `merchant`、`product`、`product_detail` 等长文本字段均为 `text`
- 已验证 merchant 与 product 测试初始化 SQL 可正常执行
